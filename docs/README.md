## Enfoque de la Documentación

Esta documentación está enfocada en:

- **Reglas de negocio**: Cómo funciona la lógica del negocio
- **Lógica condicional**: Cuándo y cómo se aplican las reglas
- **Información almacenada**: Qué datos se guardan de cada entidad
- **Interfaz de usuario**: Dónde están los botones, qué hacen, qué colores tienen, qué labels tienen
- **Flujos de trabajo**: Cómo se realizan las operaciones principales paso a paso
- **Procesos automáticos**: Qué se ejecuta automáticamente y cuándo

**NO incluye**:

- Detalles técnicos de implementación
- Ejemplos de código
- Configuración técnica
- Detalles de arquitectura
- Información irrelevante para guiar usuarios (como formatos de input, detalles de formateo automático)

## Uso para Agentes IA

Esta documentación puede ser consultada por agentes IA para:

- Entender cómo funciona la plataforma
- Saber qué información está disponible
- Conocer las reglas de negocio
- Identificar dónde están los elementos de la interfaz (botones, campos, etc.)
- Conocer los labels exactos de los botones para poder decirle al usuario que haga clic en ellos
- Comprender los flujos de trabajo paso a paso
- Entender los procesos automáticos
- Guiar a usuarios en el uso de la plataforma

## Estructura de la Documentación

Cada documento está organizado en secciones que cubren:

1. **Información almacenada**: Qué datos se guardan de cada entidad
2. **Vistas y páginas**: Dónde se encuentra cada funcionalidad
3. **Elementos de interfaz**: Botones, campos, labels exactos
4. **Flujos de trabajo**: Cómo realizar operaciones paso a paso
5. **Reglas de negocio**: Lógica condicional y reglas importantes
